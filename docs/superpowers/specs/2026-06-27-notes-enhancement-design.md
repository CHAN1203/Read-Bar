# 设计 · 笔记增强（Notes Enhancement）

日期：2026-06-27
所属：「优质沉静阅读站」之上的新增需求（独立于原四子项目；可视为子项目⑤）。

## 目标

1. 给每卷一个**自由笔记**能力（不必先选中文字）——随时记一条想法。
2. 进入一本书（封面页）时，有一处**浏览全书笔记**，按卷自动归类。
3. 修复**手机端无法划线/写批注**（触屏选词后工具条不出现）。

## 约束

- 纯静态站，GitHub Pages，零后端，自包含；不引入外部库 / CDN / 网络字体。
- 沿用冷色暗调设计系统（`--bg #13161b`，accent `--amber #d4a657`，serif 正文）。
- 保留 `prefers-reduced-motion` 与可见 focus；不破坏现有 dock / 阅读设置 / 键盘 / 进度 / 书签 / 音频 / 计时 / 编辑。
- 数据存 `localStorage`，带内存兜底。

## 决策（已与用户确认）

- 卷笔记 = **多条带时间的自由笔记**（非单一文本框），天生按卷归类。
- 封面页「总笔记」汇总 = **卷笔记 + 有批注的划线**，按卷分组（不含无批注的纯划线）。
- 手机划线工具条 = **底部固定**（窄屏）；桌面仍浮在选区旁。
- 卷笔记在**阅读时添加**；封面页只**浏览**。

## 数据模型（localStorage）

- 卷自由笔记（新键）：`readbar:vnotes:<file>` = `[ { id:String, text:String, ts:Number } ]`。
- 划线批注（已存在，不改）：`readbar:notes:<file>` = `[ {id,k,start,end,text,color,comment,ts,sec} ]`。
- `<file>` = 卷文件名去扩展名（现有约定）。

## A · 阅读页 — 卷笔记（reader.js / reader.css）

在现有「批注抽屉」里加一个**卷笔记区**（与书签区、划线列表并列；置于书签区之下、划线列表之上）：
- 一个多行输入框 + 「加笔记」按钮：点按把输入内容存为一条 `readbar:vnotes:<file>` 记录（`{id, text, ts}`），清空输入框，刷新列表。
- 列出本卷 vnotes（按 ts 倒序），每条显示文本 + 相对/简短时间，带删除按钮。
- 空态提示。

## B · 阅读页 — 手机划线修复（reader.js / reader.css）

现状：选区工具条 `.rl-tool` 仅靠 `document.addEventListener("mouseup", …)` 触发并定位到选区附近——触屏设备 `mouseup` 行为不可靠，且工具条易被选区手柄/系统菜单遮挡。

改：
1. **触发**：除 `mouseup` 外，监听 `selectionchange`（去抖）与 `touchend`，统一走同一个"有有效选区就显示工具条"的逻辑。
2. **定位**：窄屏（`matchMedia("(max-width:820px)")` 或 `.rl-tool.bottom`）时，工具条**固定在屏幕底部**、满宽、按钮触摸目标加大；桌面保持浮在选区旁。
3. 选区落在 `.col` 的 `[data-rk]` 块内才显示（沿用现有判断）。划线后清选区、隐藏工具条（沿用）。

成功标准：在窄视口（模拟手机）选中正文一句话，底部工具条出现，点一个颜色能划线，点「批注」能写批注并保存。

## C · 封面页 — 总笔记浏览（books/du-bang/index.html）

封面页是静态 HTML（不引 reader.js），用内联脚本：
1. 在进度区加一个按钮「笔记本 · N 条」（N = 全书卷笔记 + 有批注划线总数）；N=0 时按钮显示「笔记本 · 还没有笔记」且点开是空态。
2. 点击 → 打开**全屏浮层**（沿用设计系统：暗底 + 卡片）：
   - 遍历 5 卷，读 `readbar:vnotes:<volfile>` 与 `readbar:notes:<volfile>`（只取 `comment` 非空的划线）。
   - **按卷分组**：每组标题 = 卷名（如「第四卷 · 反转与楔形」）+ 一个「去该卷 →」链接（指向该卷 html）。
   - 组内：先列卷笔记（文本 + 时间），再列划线批注（引用原文 + 批注）。
   - 某卷两类都空 → 该组不显示。全空 → 浮层显示空态提示。
   - 关闭：× 按钮 / 点暗底 / `Esc`。
3. 5 卷文件名 + 标题在脚本里以小数组列出（与 ② 一致）。

## 落点（文件）

- `books/du-bang/reader.js`：卷笔记区（增/删/渲染，新键 `readbar:vnotes:<file>`）；触屏选区触发（selectionchange/touchend）+ 窄屏底部工具条切换。
- `books/du-bang/reader.css`：卷笔记区样式；`.rl-tool` 窄屏底部固定样式（含触摸目标）。
- `books/du-bang/index.html`：「笔记本」按钮 + 全屏总笔记浮层（内联脚本 + 样式）。
- `CLAUDE.md` + teaching-ebook `SKILL.md`：记新键 `readbar:vnotes:<file>` 与卷笔记/总笔记浏览/触屏划线约定。

## 验收标准

- 阅读页抽屉「卷笔记」区：输入 → 加 → 列表出现 → 刷新仍在 → 可删。
- 窄视口（模拟手机）选中正文 → 底部工具条出现 → 划线成功 → 写批注成功并保存。桌面行为不变（浮在选区旁）。
- 封面页「笔记本 · N」按钮 N 计数正确；点开浮层按卷分组，显示卷笔记 + 有批注划线，空卷不显，「去该卷」可跳转，× / 暗底 / Esc 可关。
- 不破坏现有划线/批注/进度/书签/统计；reduced-motion 下无突兀动画；本地服务器实测无 JS 报错。

## 不在本子项目内

全文检索（③）；PWA / favicon / OG / 404 / 打印 / 导出导入 / 音效渐变 / 首次引导（④）。卷笔记不做编辑（只增删，YAGNI）；总笔记不做搜索/精确跳转划线位置（保持简单）。
